## 1. Контекст

Последовательная версия алгоритма поразрядной сортировки (Radix Sort) является
эталонной реализацией, на основе которой строится вся дальнейшая работа по
распараллеливанию. Она необходима для:

1. Определения корректной логики работы с вещественными числами (обработка
   знака).
2. Создания базовой линии производительности (baseline), относительно которой
   будет рассчитываться ускорение ($S$) и эффективность ($E$) в параллельных
   версиях.

## 2. Постановка задачи

- **Вход:** вектор с вещественными числами типа double размера N;

- **Выход:** упорядоченный по возрастанию вектор с вещественными числами типа
  double размера N;

- **Ограничения:**
- Критерий корректности: выходной вектор $result$ считается отсортированным
  корректно, если для любых индексов $i$ и $j$ в диапазоне $[0, n-1]$
  выполняется условие: $$i < j \implies result_i \leq result_j$$

Это означает, что для любого элемента (кроме последнего) его значение не
превышает значения следующего за ним элемента, что гарантирует соблюдение
порядка неубывания во всем векторе.

## 3. Базовый алгоритм

Алгоритм основан на побайтовом распределении чисел по 256 корзинам.

1. **Преобразование (InBytes):** каждое число `double` интерпретируется как
   `uint64_t`. Если число положительное, инвертируется только знаковый бит. Если
   отрицательное — инвертируются все биты. Это позволяет использовать
   стандартный лексикографический порядок байтов для сортировки.
2. **Проходы по байтам:** выполняется 8 итераций (для каждого из 8 байтов
   64-битного числа). На каждой итерации:
    - Подсчитывается количество вхождений каждого значения байта (0–255).
    - Вычисляются префиксные суммы для определения позиций в выходном буфере.
    - Элементы переносятся во вспомогательный массив согласно значению текущего
      байта.
3. **Обратное преобразование (FromBytes):** после 8 проходов выполняется
   инверсия битов в обратном порядке для восстановления исходных значений
   `double`.

**Асимптотика:**

- **По времени:** $O(8 \cdot N) \approx O(N)$. Алгоритм линеен относительно
  размера входных данных.
- **По памяти:** $O(N)$. Требуется создание двух вспомогательных массивов типа
  `uint64_t` для хранения промежуточных результатов.

**Инвариант:** после $k$-го прохода массив отсортирован по первым $k$ младшим
байтам. **Критерий корректности:** для любых $i < j$ выполняется
$A[i] \le A[j]$.

## 4. Детали реализации

В рамках архитектуры `BaseTask` используются стандартные методы
`ValidationImpl`, `PreProcessingImpl` и `PostProcessingImpl` для формальной
проверки и подготовки векторов. Основная логика сортировки сосредоточена в
методе `RunImpl` и наборе специализированных функций:

- **RunImpl:** метод, организующий поток данных. Он извлекает исходный массив из
  `GetInput()`, вызывает функцию `RadixSort` для выполнения поразрядной
  обработки и по завершении копирует отсортированные данные в `GetOutput()`.
- **InBytes(double d):** метод для работы с вещественными числами. Он
  преобразует `double` в `uint64_t` с помощью `memcpy`. Для соблюдения
  правильного порядка при сортировке применяется битовая маска
  `0x8000000000000000`:
  - если число положительное, инвертируется только знаковый бит.
  - если число отрицательное, инвертируются все биты.
- **FromBytes(uint64_t bits):** выполняет обратную операцию: восстанавливает
  исходный битовый паттерн IEEE 754, учитывая инверсии, сделанные в `InBytes`, и
  преобразует его обратно в тип `double`.
- **SortByByte(...):** метод, реализующий один проход сортировки (Counting Sort)
  по одному конкретному байту (от 0 до 7).
    1. С помощью `reinterpret_cast<unsigned char*>` массив просматривается как
       последовательность байтов, и в `std::array<int, 256>` подсчитывается
       количество вхождений каждого значения.
    2. Частоты преобразуются в начальные индексы (смещения) для каждой
       «корзины».
    3. Элементы копируются из `bytes` в `out` на вычисленные позиции,
       обеспечивая стабильность сортировки.
- **RadixSort(...):** создает два временных буфера типа `uint64_t` и запускает
  цикл из 8 последовательных вызовов `SortByByte`. Реализована техника двойной
  буферизации: на каждом шаге исходный массив и целевой массив меняются местами
  (`src_ptr` и `dst_ptr`), что исключает лишние копирования данных между
  проходами.

```cpp
void PapulinaYRadixSortSEQ::RadixSort(double *arr, int size) {
  std::vector<uint64_t> bytes(size), out(size);
  for (int i = 0; i < size; i++) bytes[i] = InBytes(arr[i]);

  uint64_t *src_ptr = bytes.data();
  uint64_t *dst_ptr = out.data();

  for (int byte = 0; byte < 8; byte++) {
    SortByByte(src_ptr, dst_ptr, byte, size);
    std::swap(src_ptr, dst_ptr);
  }

  for (int i = 0; i < size; i++) arr[i] = FromBytes(src_ptr[i]);
}
```

## 5. Проверка корректности

Корректность последовательной версии проверялась путем сравнения выходных данных
с результатом `std::sort` на расширенном наборе из 7 функциональных тестов:

1. **test1:** положительные числа с дробной частью.
2. **test2:** отрицательные числа (проверка корректности работы битовой маски
   для знака).
3. **test3:** смешанные знаки и наличие нуля (проверка правильного расположения
   нуля относительно положительных и отрицательных чисел).
4. **test4:** числа с большой точностью и разным количеством знаков после
   запятой.
5. **test5:** группа очень близких значений (проверка чувствительности
   сортировки к младшим разрядам/байтам).
6. **test6:** экстремально малые значения (близкие к машинному нулю).
7. **test7:** массив идентичных элементов.
